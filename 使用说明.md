## 1. 系统简介

本系统是一个**纯网页端实验记录工具**，用于记录芯片/器件测试过程中各批次样本在不同平台、条件下的 Pass/Fail 结果。

**核心概念：**

| 名称 | 含义 | 示例 |
|------|------|------|
| **分类（批次）** | 一组待测样本的批次标识 | 批次A、LOT-2026Q1 |
| **ic（样本编号）** | 每批次内的样本序号 | 1 ~ N（每批次可独立设置数量） |
| **平台** | 测试平台/系统 | Android、iOS、Windows |
| **条件** | 每个平台下的测试环境/参数 | 常温25°C、高温85°C（各平台独立） |
| **项目** | 每个平台下的测试指标 | 电流测试、功能验证（各平台独立） |
| **P / F** | 测试结果 | 🟢 P = Pass（通过），🟠 F = Fail（失败） |

**特点：**
- ✅ 无需安装，浏览器直接打开
- ✅ 数据持久化存储（通过本地服务器写入 `data.json`，降级时存 localStorage）
- ✅ 支持多平台、每平台独立条件和测试项目
- ✅ 支持多次测试结果（第1次/第2次重测），每次独立统计 Pass Rate
- ✅ 字段拖拽排序、隐藏/显示
- ✅ 多维筛选（结果类型、分类、平台、条件、项目、次数）
- ✅ 一键导出 Excel（含统计汇总），支持导入 Excel 恢复数据
- ✅ 固定横向滚动条，宽表格操作友好

## 2. 启动方式

### 推荐方式：本地服务器（双击启动）

1. 确保已安装 [Node.js](https://nodejs.org/)
2. 双击项目目录中的 **`启动.bat`**
3. 浏览器自动打开 **http://localhost:3030**

> 💡 关闭 `启动.bat` 窗口即停止服务器。若服务已在运行，双击只会打开浏览器。

### 手动启动

```bash
cd d:\workspace\WorkBuddy\LabRecord
node server.js
```

浏览器访问：**http://localhost:3030**

> ⚠️ 若直接双击 `index.html`（file:// 协议），数据将存入 localStorage 而非 `data.json`，重启服务器后数据不会自动同步。**强烈建议使用服务器模式。**

## 4. 快速上手：第一次使用

### Step 1：设置实验字段

点击右上角 **「⚙ 管理字段」**：

1. **分类（批次）**：输入批次名称，按 Enter 添加
2. **平台**：添加测试平台（如 `Android`、`iOS`），每个平台独立展开配置
3. **条件**：在各平台 block 内添加测试条件
4. **项目**：在各平台 block 内添加测试项目
5. 点击 **「✓ 应用并关闭」**，表格自动生成

> 💡 支持拖拽 ⠿ 排序，👁 图标可隐藏/显示字段

### Step 2：填写实验信息

在顶部工具栏填写**实验名称**和**备注**（可选）。

### Step 3：录入测试结果

点击表格中任意数据单元格 → 弹出录入窗口 → 选择 Pass / Fail → 确认。

### Step 4：保存

点击 **「💾 保存」**，记录保存到左侧历史列表。

## 5. 录入测试结果

### 基本操作

1. 点击表格数据格（显示 `＋` 的区域）
2. 弹出录入窗口，显示当前位置（批次/样本/平台/条件/项目）
3. 选择结果：🟢 **Pass** 或 🟠 **Fail**
4. 可选填写**备注**
5. 点击 **「确认添加」**

### 多次结果

同一格可录入多次（如重测），每次叠加显示：
```
[P] [F] [P]   ← 三次结果
```

表格按次数**纵向对齐**：第1次各项目在同一行，第2次各项目在下一行，互不干扰。

### 删除单次结果

悬停结果标签，点击标签内的 **✕**。

### 总结列

每个条件组末尾的「**总结**」列：
- 显示该次各项目**全部 Pass** 才算 Pass
- 底部 **PR 第N次** 行显示每次的通过率
- **总计** 行（颜色更深）显示所有次数的汇总通过率

## 6. 管理字段

点击 **「⚙ 管理字段」** 随时修改。

### 平台管理

每个平台是独立的 block，包含：
- 平台名称（双击编辑）
- 该平台下的**条件列表**（独立增删改、拖拽排序、👁 隐藏）
- 该平台下的**项目列表**（独立增删改、拖拽排序、👁 隐藏）

> ⚠️ 删除有测试数据的条件或项目时，系统会弹出确认提示，防止误删。

### 分类管理

可增删分类（批次），每批次样本数量独立设置。

### 字段隐藏

点击字段旁的 👁 图标可临时隐藏，不影响数据，隐藏字段的数据仍保留。

### 拖拽排序

所有字段（分类/平台/条件/项目）均支持拖拽 ⠿ 排序，排序后数据 key 自动重映射。

## 7. 保存与历史记录

### 保存当前实验

1. 填写**实验名称**（必填）
2. 点击 **「💾 保存」**
3. 左侧边栏出现该记录

### 加载历史记录

点击左侧侧边栏任意历史记录 → 自动加载。

### 复制结构

悬停历史记录 → 点击 **「复制结构」**：创建新实验，保留字段配置，**清空所有数据**（适合下一轮实验）。

### 删除记录

悬停历史记录 → 点击红色 **「删除」** → 确认。

> 💡 侧边栏可通过左上角箭头收起，节省屏幕空间。

## 8. 搜索与筛选

筛选栏支持多维组合筛选：

| 筛选器 | 说明 |
|--------|------|
| **文本搜索** | 实时搜索批次/样本/备注内容 |
| **结果类型** | Pass（有Pass）/ Fail（有Fail）/ 全Pass / 全Fail |
| **分类** | 多选批次 |
| **平台** | 多选平台 |
| **条件** | 多选条件（汇总所有平台） |
| **项目** | 多选项目 |
| **次数** | 多选第N次（如只看第2次重测） |

所有多选筛选均为自定义下拉组件，支持全选/清空。

## 9. 导出 / 导入 Excel

### 导出

点击 **「⬇ 导出 Excel」** → 下载 `.xlsx` 文件。

文件名：`实验名称_日期.xlsx`

| Sheet | 内容 |
|-------|------|
| **实验数据** | 完整原始数据，表头格式：`平台\|条件\|项目` |
| **统计汇总** | 按「批次 × 平台 × 条件」汇总的 Pass/Fail 数量和通过率 |

### 导入

点击 **「📥 导入 Excel」** → 选择之前导出的文件。

- 支持新格式（`平台|条件|项目`）和旧格式（`条件 - 项目`）
- 导入后各平台独立存储条件和项目

## 10. 统计分析

点击顶部 **「📊 统计」** 切换视图。

### 统计卡片

显示当前实验总体数据：总 Pass / 总 Fail / 整体通过率 / 批次数 / 条件数 / 项目数。

### 四张图表

| 图表 | 说明 |
|------|------|
| **各批次通过率** | 柱状图，绿≥80%，橙50-80%，红<50% |
| **各条件 Pass/Fail 分布** | 对比柱状图，直观看哪个条件失败多 |
| **各测试项目通过情况** | 横向柱状图，找出最容易失败的项目 |
| **Pass/Fail 总览** | 环形饼图，整体通过/失败比例 |

## 11. 常见问题

**Q：关闭浏览器后数据还在吗？**
> A：在。服务器模式下数据存在 `data.json`，重启服务器后仍可访问。若使用 file:// 直接打开，数据在 localStorage，清缓存会丢失，建议定期导出 Excel。

**Q：数据存在哪里？**
> A：服务器模式（推荐）：`data.json` 文件，和 `index.html` 同目录。直接打开模式：浏览器 localStorage，key 为 `labrecord_v2`。

**Q：每批次样本数量能改吗？**
> A：可以。在「管理字段」中每个分类旁边有样本数输入框，独立设置。

**Q：能否修改已录入结果？**
> A：可以。点击单元格上的 ✕ 删除错误记录，再重新录入。

**Q：导出的 Excel 没有颜色？**
> A：Excel 导出为文字格式（P/F），暂不含颜色。如需彩色，可在 Excel 中使用条件格式。

**Q：表格太宽，横向滚动不方便？**
> A：页面底部有固定横向滚动条，随时可见，无需滚到表格最底部。

**Q：怎么备份全部历史？**
> A：服务器模式直接备份 `data.json`。localStorage 模式：F12 → Application → localStorage → `labrecord_v2` 复制 JSON。

**Q：旧版数据（没有平台字段）能用吗？**
> A：可以，加载旧数据时自动迁移：创建默认平台并将旧数据映射到该平台下。
