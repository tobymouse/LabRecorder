## 1. 系统简介

一个专为 **IC / 器件 validation** 设计的网页端测试记录工具，用于管理多批次样本在不同平台、测试条件下的 Pass/Fail 矩阵，并提供实时统计与 Redmine 集成。

**核心概念：**

| 名称 | 含义 | IC Validation 示例 |
|------|------|-------------------|
| **分类（批次/Lot）** | 一组待测样品的批次标识 | EVT-A、EVT-B、DVT-01 |
| **ic（样本编号）** | 每批次内的样本序号 | 1 ~ N 片（每批次独立设置） |
| **平台** | 测试平台 / 封装形态 | Socket/COB（裸片/封装二选一）、整机 |
| **条件** | 平台下的测试环境参数 | Normal 25°C、High Temp 85°C、Low Temp -40°C（各平台独立） |
| **项目** | 平台下的测试指标 | Current (sleep)、RSSI、Throughput（各平台独立） |
| **P / F** | 测试结果 | 🟢 P = Pass（通过），🟠 F = Fail（失败） |

**主要特点：**
- ✅ 无需安装，浏览器直接打开（需 Node.js 提供本地服务器）
- ✅ 数据持久化存储（`data.json`，降级时存 localStorage）
- ✅ 每个平台独立配置条件和测试项目
- ✅ 支持多次测试（重测），每次独立统计 Pass Rate
- ✅ 字段拖拽排序、隐藏/显示
- ✅ 多维筛选（结果类型、分类、平台、条件、项目、次数）
- ✅ 导出 / 导入 Excel（含统计汇总 Sheet）
- ✅ 固定横向滚动条，宽表格友好
- ✅ Redmine 集成：关联 Issue + 一键同步测试简报

## 2. 启动方式

### 推荐：双击启动（Windows）

1. 确保已安装 [Node.js 18+](https://nodejs.org/)
2. 双击项目目录中的 **`启动.bat`**
3. 浏览器自动打开 **http://localhost:3030**

> 💡 关闭 `启动.bat` 窗口即停止服务器。若服务已在运行，双击只会打开浏览器。

### 手动启动

```bash
cd /path/to/lab-recorder
node server.js
```

浏览器访问：**http://localhost:3030**

> ⚠️ 若直接双击 `index.html`（file:// 协议），数据将存入 localStorage 而非 `data.json`，重启服务器后数据不会同步。**强烈建议使用服务器模式。**

## 4. 快速上手

以蓝牙 SoC 验证为例：

### Step 1：配置字段

点击 **「⚙ 管理字段」**：

1. **分类（批次）**：添加 `EVT-A`、`EVT-B`，每批次设置样本数（如 10 片）
2. **平台**：添加 `Socket/COB`、`整机`
3. **Socket/COB** 展开 → 添加条件：`Normal 25°C` / `High Temp 85°C` / `Low Temp -40°C`
4. **Socket/COB** → 添加项目：`Current (sleep)` / `Current (TX)` / `RSSI` / `Throughput`
5. **整机** → 添加条件：`Normal 25°C`；项目：`Pairing` / `OTA` / `功能测试`
6. 点击 **「✓ 应用并关闭」**，测试矩阵自动生成

> 💡 支持拖拽 ⠿ 排序，👁 图标可临时隐藏字段

### Step 2：填写实验信息

在顶部填写**实验名称**、**备注**，可调整**目标通过率**和**告警通过率**阈值。

### Step 3：录入测试结果

点击表格任意数据格 → 弹出录入窗口 → 选择 Pass / Fail → 填备注（可选）→ 确认。

### Step 4：保存

点击 **「💾 保存」**，记录保存到左侧历史列表。

## 5. 录入测试结果

### 基本操作

1. 点击数据格（显示 `＋` 的区域）
2. 弹窗显示当前位置（批次/样本/平台/条件/项目）
3. 选择 🟢 **Pass** 或 🟠 **Fail**
4. 可选填写**备注**（如 "电流 12mA，spec 10mA"）
5. 点击 **「确认添加」**

### 多次测试（重测）

同一格可录多次，每次叠加显示：

```
[P] [F] [P]   ← 第1次、第2次（重测）、第3次
```

表格按次数**纵向对齐**：第1次各项目在同一行，第2次各项目在下一行，独立统计。

### 删除单次结果

悬停结果标签 → 点击标签内的 **✕**。

### 总结列

每个条件组末尾有「**总结**」列：
- 该次**所有项目全 Pass** 才显示 Pass
- 底部 **PR 第N次** 行显示该次通过率
- **总计** 行（颜色加深）显示所有次数汇总

## 6. 管理字段

点击 **「⚙ 管理字段」** 随时修改，改完点「✓ 应用并关闭」生效。

### 平台管理

每个平台是独立 block，包含：
- **平台名称**（双击编辑）
- **条件列表**：独立增删改、拖拽 ⠿ 排序、👁 隐藏
- **项目列表**：独立增删改、拖拽 ⠿ 排序、👁 隐藏

> ⚠️ 删除有测试数据的条件或项目时，系统会弹出确认提示，防止误删。

### 分类管理

增删分类（批次），每批次样本数量独立设置。

### 字段隐藏

点 👁 图标临时隐藏，隐藏不影响数据，随时可恢复。

### 拖拽排序

所有字段均支持拖拽 ⠿ 排序，排序后 data key 自动重映射，不会乱数据。

## 7. 保存与历史记录

### 保存当前实验

1. 填写**实验名称**
2. 点击 **「💾 保存」**
3. 左侧侧边栏出现该记录

### 加载历史记录

点击左侧侧边栏任意历史记录 → 自动加载。

### 复制结构

悬停历史记录 → 「复制结构」：创建新实验，**保留字段配置，清空所有数据**（适合开始新一轮测试）。

### 删除记录

悬停历史记录 → 红色「删除」→ 确认。

> 💡 侧边栏可通过左上角箭头收起，节省屏幕空间。

---

## 8. 搜索与筛选

筛选栏支持多维组合筛选，全部为多选下拉组件：

| 筛选器 | 说明 |
|--------|------|
| **文本搜索** | 实时过滤批次/备注内容 |
| **结果类型** | Pass（有Pass）/ Fail（有Fail）/ 全Pass / 全Fail |
| **分类** | 多选批次（如只看 DVT-01） |
| **平台** | 多选平台 |
| **条件** | 多选条件（汇总所有平台去重） |
| **项目** | 多选项目 |
| **次数** | 多选第N次（如只看第2次重测） |

所有多选筛选支持**全选**和**清空**，右侧「✕ 清除」一键重置所有筛选。

## 9. 导出 / 导入 Excel

### 导出

点击 **「⬇ 导出 Excel」** → 下载 `.xlsx` 文件（文件名：`实验名称_日期.xlsx`）

| Sheet | 内容 |
|-------|------|
| **实验数据** | 完整原始矩阵，表头格式：`平台\|条件\|项目` |
| **统计汇总** | 按「批次 × 平台 × 条件」聚合的 Pass/Fail 计数和通过率 |

### 导入

点击 **「📥 导入 Excel」** → 选择导出的文件。

- 支持新格式（`平台|条件|项目`）和旧格式（`条件 - 项目`）
- 导入后各平台独立存储条件和项目

## 10. 统计分析

点击顶部 **「📊 统计分析」** Tab 切换视图。

### 独立筛选栏

统计视图有自己的筛选栏（分类 / 平台 / 条件），与表格视图独立，不互相干扰。

### 统计卡片

总 Pass / 总 Fail / 整体通过率 / 批次数 / 条件数 / 项目数。

### 四张图表

| 图表 | 说明 |
|------|------|
| **各批次通过率** | 柱状图，绿≥目标PR，橙≥告警PR，红<告警PR |
| **各条件 Pass/Fail 分布** | 对比柱状图，快速定位哪个温度/电压失败最多 |
| **各项目通过情况** | 横向柱状图，找出最容易失败的测试项目 |
| **Pass/Fail 总览** | 环形饼图，整体通过/失败比例 |

## 11. Redmine 集成

### 配置

首次使用点击顶部 **「🐛 Redmine」** 配置按钮，填入：
- Redmine URL（如 `https://redmine.yourcompany.com`）
- API Key（从 Redmine 用户设置页获取）
- 项目 ID 和 Tracker ID

配置保存在浏览器 localStorage，刷新不丢失。

### 关联 Issue

点击 **「🐛 提 Issue」**：

1. 弹窗顶部显示**已关联 Issue 列表**（可点击跳转，可 ✕ 取消关联）
2. 搜索框搜索已有 Issue 并关联
3. 或填写标题/描述**新建 Issue**（自动关联到当前实验）

Issue 关联是实验级别的，不绑定单个单元格。

### 同步简报

当实验有关联 Issue 时，顶部出现 **「📤 同步简报」** 按钮。

点击后自动生成测试摘要并追加到所有关联 Issue 的 journal：

```
【测试简报同步】
整体通过率：85% (170/200)

各平台通过率：
- Socket/COB: 88%
- 整机: 92%

各批次通过率：
- EVT-A: 90%
- EVT-B: 80%

Fail 清单：
- EVT-A #3 / Socket/COB / High Temp 85°C / Current (TX): 电流偏高 12mA vs spec 10mA
- EVT-B #7 / Socket/COB / Normal 25°C / Throughput: 低于 spec
...
```

## 12. 常见问题

**Q：关闭浏览器后数据还在吗？**
> 在。服务器模式下数据存在 `data.json`，重启服务器后仍可访问。建议定期导出 Excel 备份。

**Q：数据存在哪里？**
> 服务器模式（推荐）：`data.json`，和 `index.html` 同目录。备份直接复制这个文件即可。

**Q：每批次 IC 数量能改吗？**
> 可以。在「管理字段」中每个分类旁有样本数输入框，独立设置。

**Q：能修改已录入的结果吗？**
> 可以。点击单元格上的 ✕ 删除错误结果，再重新录入。

**Q：旧版数据（没有平台字段）能用吗？**
> 可以，加载时自动迁移到新数据结构。

**Q：导出的 Excel 没有颜色？**
> Excel 导出为文字格式（P/F），暂不含颜色。可在 Excel 中用条件格式自行着色。

**Q：表格太宽，横向滚动不方便？**
> 页面底部有固定横向滚动条，随时可见，不需要滚到页面末尾。

**Q：Redmine 连不上怎么办？**
> 检查 URL 是否包含协议头（`https://`），API Key 是否正确，以及服务器是否能访问 Redmine（Redmine 请求由 server.js 代理）。

**Q：怎么备份全部历史？**
> 服务器模式：直接复制 `data.json`。如需人类可读格式，点「⬇ 导出 Excel」逐条导出。
